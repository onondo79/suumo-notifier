# SUUMO 新着賃貸物件 通知ツール

SUUMOの検索結果（錦糸町・押上・住吉）を1日4回チェックし、
**前回から新しく増えた物件だけ**を、エリアごとにGmailで通知するツールです。

---

## 各ファイルの役割

| ファイル | 役割 |
|---|---|
| `suumo_notifier.py` | 本体。物件を取得→前回と比較→メール送信→記録を保存する。 |
| `requirements.txt` | 必要な外部ライブラリの一覧。 |
| `.env.example` | パスワード設定の「見本」。コピーして `.env` を作る。 |
| `.env` | 自分のGmailアドレスとアプリパスワードを書くファイル（**Gitに上げない**）。 |
| `.gitignore` | Gitに上げないファイルの指定（`.env` を含む）。 |
| `data/` | エリアごとの記録（`seen_*.json`）を保存する場所。新着判定に使う。 |
| `.github/workflows/notify.yml` | GitHub Actionsの自動実行設定（JST 9:03/15:03/18:03/22:03）。 |

> **用語メモ**
> - **ライブラリ**: 他の人が作った便利な機能のまとまり。
> - **環境変数 / Secrets**: パスワードなどをコードに直接書かずに渡す仕組み。
> - **コミット / プッシュ**: 変更を記録し、GitHubに反映させること。

---

## 動作の仕組み（新着判定）

- 物件ごとに、URLに含まれる識別子（`jnc_数字`）を「キー」として記録します。
- 今回のキー一覧と、前回保存したキー一覧を比べ、**前回に無かったもの**を新着とします。
- 記録はエリアごとに別ファイル（`data/seen_kinshicho.json` など）で保持します。
- **初回実行**は、記録を作るだけで全件を新着扱いにはしません。
  「初回登録」の旨をメールで通知します。
- 新着が0件のエリアも「新着なし」というメールを必ず送ります（毎回3通）。

## 1. ローカルPC（自分のパソコン）でテストする

### 手順1: Pythonが入っているか確認
PowerShellで次を実行します。バージョン（例: `Python 3.12.x`）が表示されればOKです。
```powershell
py --version
```
> このPCでは `python` ではなく **`py`** コマンドを使います
> （`python` はWindowsストアのダミーを指しているため）。

### 手順2: このフォルダへ移動
```powershell
cd C:\Users\yuyuo\suumo-notifier
```

### 手順3: 必要なライブラリをインストール
```powershell
py -m pip install -r requirements.txt
```

### 手順4: Gmailの「アプリパスワード」を取得
1. Googleアカウントで**2段階認証**を有効にする（必須）。
   https://myaccount.google.com/security
2. 「アプリ パスワード」のページを開く。
   https://myaccount.google.com/apppasswords
3. 名前を適当に（例: `suumo`）付けて作成すると、**16文字のパスワード**が表示される。
   これをコピーする（普段のログインパスワードとは別物です）。

### 手順5: `.env` ファイルを作る
`.env.example` をコピーして `.env` という名前に変え、中身を自分の情報に書き換えます。
```powershell
copy .env.example .env
notepad .env
```
メモ帳が開くので、次のように書き換えて保存します（スペースは入れない）。
```
GMAIL_ADDRESS=jibun@gmail.com
GMAIL_APP_PASSWORD=取得した16文字
```

### 手順6: 実行してメールを確認
```powershell
py suumo_notifier.py
```
- 初回なので、各エリア「初回登録」のメールが**3通**届けば成功です。
- `data/` フォルダに `seen_*.json` が3つできているはずです。
- もう一度すぐ実行すると、今度は「新着なし」のメールが3通届きます
  （前回との差がないため）。これで新着判定も動いていることを確認できます。

> **届かないとき**
> - 迷惑メールフォルダを確認。
> - `GMAIL_APP_PASSWORD` は16文字を続けて入力（スペースなし）。
> - 2段階認証が有効になっているか確認。

## 2. GitHubにアップロードして自動実行する

詳しい手順は、このあとチャットで1ステップずつ案内します。概要は次のとおりです。

1. GitHubで新しいリポジトリを作る（**Privateを推奨**）。
2. このフォルダをGitに登録してpushする。
3. リポジトリの **Settings → Secrets and variables → Actions** で、
   秘密情報を2つ登録する。

   | Secret名 | 入れる値 |
   |---|---|
   | `GMAIL_ADDRESS` | 自分のGmailアドレス |
   | `GMAIL_APP_PASSWORD` | 取得した16文字のアプリパスワード |

4. **Actions** タブから手動実行（Run workflow）で動作確認する。
5. あとは設定した時刻（JST 9:03 / 15:03 / 18:03 / 22:03）に自動で動きます。

> **無料枠について**
> 1回の実行は数十秒程度、1日4回でも月の利用時間はごくわずかなので、
> GitHub Actionsの無料枠（パブリックは無制限、プライベートは月2,000分）に十分収まります。
